# AGENTS.md

本文件为在本仓库中工作的 AI 编码代理提供项目约定。修改代码前先阅读
`README.md`，涉及系统边界或接口时同时查阅 `docs/architecture.md`、
`docs/openapi.yaml` 和 `docs/frontend-api.md`。

## 项目概览

Block Beast Platform 是 Go 后端工程，采用模块化单体，并提供三个独立进程：

- `cmd/api`：HTTP API、鉴权和管理端入口。
- `cmd/worker`：结算、事件消费及异步任务。
- `cmd/realtime`：实时连接网关。

核心代码位于 `internal/`：

- `application/`：应用服务和用例编排。
- `domain/`：领域模型、规则及仓储接口或实现。
- `platform/`：HTTP、NATS JetStream 等基础设施适配。
- `config/`：环境变量和运行配置。

保持 `identity`、`wallet`、`game`、`agent`、`realtime`、`chain` 和
`operations` 的领域边界。跨领域流程应由应用层编排，不要把业务规则放入
HTTP handler、进程入口或消息传输代码。

## 开发与验证

项目使用 Go 1.26.5。提交修改前至少运行：

```sh
gofmt -w <修改的 Go 文件>
go test ./...
```

如修改了依赖，再运行：

```sh
go mod tidy
```

本地依赖服务通过 Docker Compose 启动：

```sh
docker compose up --build
```

不要默认删除数据卷；`docker compose down --volumes` 仅在用户明确要求清空
本地数据时使用。

## 编码约定

- 遵循 Go 惯例，保持包职责单一，错误应包含有助于定位问题的上下文。
- 优先使用标准库；引入新依赖前确认现有代码或标准库无法合理满足需求。
- 金额使用最小货币单位的整数表示，不使用浮点数。
- 所有会被重试的写操作必须考虑幂等性。
- 钱包余额、账本记录和相关 outbox 事件应在同一数据库事务中提交。
- 领域事件先写入 outbox；不要在数据库事务中直接依赖外部消息发布成功。
- 新增或修改行为时补充同包测试，覆盖成功、失败、权限和重复请求等关键路径。
- 不要无故改变既有 API 的字段名、状态码或错误语义；接口变化需同步更新
  `docs/openapi.yaml` 和 `docs/frontend-api.md`。

## 数据库迁移

- 迁移文件位于 `migrations/`，只新增按序编号的迁移，不修改已发布迁移。
- DDL 应尽可能兼容滚动部署，并明确约束、索引和默认值。
- 涉及余额、账本、投注、结算、充值或提现的数据修改必须保持事务一致性，
  并考虑并发和重复投递。

## 配置与安全

- 不要提交 `.env`、私钥、令牌、密码、生产地址或其他敏感信息。
- 新增配置项时同步更新 `.env.example`，并提供安全的本地默认行为或清晰报错。
- 认证、资金、链上回调和后台操作属于高风险路径；修改时保留或增强鉴权、
  验签、审计和幂等保护。
- 日志中不得输出密码、完整访问令牌、私钥或其他可复用凭据。

## 修改范围

- 保留用户已有的未提交修改，不重置或覆盖无关文件。
- 只修改完成当前任务所需的文件，避免顺手进行大范围重构。
- 若代码行为与文档不一致，先确认实际约束；修复后同步代码、测试和文档。
